# Chamber Appointment System v1.0.75

## Booking profile order improvement

- Moved **Booking for** (self/family patient profile) to the first booking step.
- It now appears immediately below the selected doctor and **before** the appointment date.
- The **Add a relative** option is kept beside the Booking for field, allowing a new family member to be added before selecting the date.
- Simplified booking progress to **Details → Serial → Confirm**.
- The confirmation screen Change actions return to the appropriate current step.
- Existing appointment validation, one-active-appointment rule, quick relative addition, reconfirmation lock, and backend workflows are unchanged.
