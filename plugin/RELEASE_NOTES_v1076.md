# Version 1.0.76 — Single Serial Selection + Current Status

- Fixed the frontend booking flow so tapping one serial does not reload the serial grid or clear the selection.
- Removed the date-field blur reload that was triggered when a patient tapped a serial on mobile.
- **Review booking** is now disabled until a serial is selected, then enables immediately after the first selection.
- The backend appointment worklist now labels the appointment status column as **Current Status**.
