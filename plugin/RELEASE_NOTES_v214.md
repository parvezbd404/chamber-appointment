# Chamber Appointment System 2.1.4

- Made the backend **Add to Waiting List** action available in both Live and Development modes.
- Chamber staff can manually add an active patient even when normal appointment slots are still available.
- Development Mode suppresses real waiting-list SMS notifications.
- Live Mode follows the configured waiting-list SMS notification workflow.
- Preserved capability, nonce, doctor-access, active-patient, date, and duplicate-entry validation.
- Waiting-list promotion remains available in both application modes.
